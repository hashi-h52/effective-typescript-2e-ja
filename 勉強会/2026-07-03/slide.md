# 導入1 - 書籍の紹介

[Effective TypeScript 第2版](https://www.oreilly.co.jp/books/9784814401093/)

## どんな本か

- 独立した小さな「項目」の集まり形式
- 各項目の要点が明快 → 気になる項目から拾い読みでOK
- 文法の基礎知識から、実践的な設計パターンまで幅広く収録
- 正直、本を買わなくても[公式リポジトリ](https://github.com/oreilly-japan/effective-typescript-2e-ja) + AIで十分理解できる

# 導入2 - なぜ読むか（1/2）

- コーディング・設計をAIが担っても、最終承認は人間の仕事
- AIは既存実装に引っ張られやすく、既存コードのダメな部分も再生産してしまう
- 再生産を防ぐには、既存コードからは得られない知識を人間が持つ必要がある

# 導入2 - なぜ読むか（2/2）

## フロントエンド力の分解

- コンポーネント設計：ビジュアルから起こせる → AIが得意
- ライブラリの使い方（RHF, TanStack Queryなど）→ AIが詳しい
- TSの型定義 = 仕様のモデリング：仕様や拡張性の理解が必要 → AIに丸投げ不可
